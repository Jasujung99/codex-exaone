# 한국어 교사용 수업자료 제작 도구 샘플 프롬프트 세트

이 문서는 FriendliAI의 EXAONE 계열 모델을 활용한 한국어 교사용 수업자료 제작 MVP에서 사용할 수 있는 샘플 프롬프트를 정리한다. 실제 구현에서는 사용자가 입력한 폼 값을 변수로 주입하고, 공통 시스템 프롬프트와 콘텐츠 유형별 사용자 프롬프트를 조합한다.

## 1. 공통 시스템 프롬프트

```text
당신은 외국어로서의 한국어 교육 전문가입니다.
한국어 교사가 수업에서 사용할 수 있는 교육 자료를 만듭니다.

반드시 지켜야 할 원칙:
1. 학습자 수준에 맞는 어휘와 문법을 사용합니다.
2. 요청한 목표 문법과 목표 어휘를 자연스럽게 포함합니다.
3. 한국어 표현은 자연스럽고 명확해야 합니다.
4. 문제의 정답은 하나로 명확해야 합니다.
5. 문화적 편견, 성별 고정관념, 차별적 표현을 피합니다.
6. 교사가 바로 복사해 사용할 수 있도록 구조화된 형식으로 작성합니다.
7. 모르는 사실을 지어내지 말고, 일반적인 한국어 교육 맥락에서 안전한 예시를 사용합니다.
8. 출력은 요청한 형식을 따릅니다.
```

## 2. 공통 변수

프롬프트 템플릿에서 사용할 기본 변수는 다음과 같다.

```text
{level}: 초급, 중급, 고급, TOPIK I, TOPIK II
{topic}: 수업 주제
{target_grammar}: 목표 문법
{target_vocabulary}: 목표 어휘
{lesson_duration}: 수업 시간
{exercise_count}: 문제 수
{include_answers}: 정답 포함 여부
{include_explanations}: 해설 포함 여부
{include_teacher_tips}: 교사용 팁 포함 여부
{output_language}: 출력 언어
```

## 3. 대화문 수업자료 생성 프롬프트

### 목적

상황 중심 한국어 수업에서 사용할 대화문, 어휘, 문법, 연습문제를 생성한다.

### 사용자 프롬프트 템플릿

```text
다음 조건에 맞는 한국어 수업자료를 만들어 주세요.

조건:
- 학습자 수준: {level}
- 주제: {topic}
- 목표 문법: {target_grammar}
- 목표 어휘: {target_vocabulary}
- 수업 시간: {lesson_duration}
- 연습문제 수: {exercise_count}
- 정답 포함: {include_answers}
- 해설 포함: {include_explanations}
- 교사용 팁 포함: {include_teacher_tips}

출력 형식:
# 제목

## 학습 목표
- 2~3개 작성

## 도입 질문
- 수업 시작 전에 사용할 질문 3개

## 대화문
- 등장인물 2명
- 학습자 수준에 맞는 길이
- 목표 문법을 자연스럽게 3회 이상 포함

## 핵심 어휘
| 어휘 | 뜻 | 예문 |
| --- | --- | --- |

## 문법 포인트
- 목표 문법의 의미
- 사용 상황
- 형태 정보
- 예문 3개

## 연습문제
- 객관식, 빈칸 채우기, 짧은 답변 문제를 섞어서 작성

## 정답 및 해설
- 각 문제의 정답과 간단한 해설

## 교사용 활용 팁
- 짝 활동 또는 역할극 아이디어 포함
```

## 4. 읽기 지문 생성 프롬프트

### 목적

수준별 읽기 수업에 사용할 짧은 지문과 독해 문제를 생성한다.

### 사용자 프롬프트 템플릿

```text
다음 조건에 맞는 한국어 읽기 수업자료를 만들어 주세요.

조건:
- 학습자 수준: {level}
- 주제: {topic}
- 목표 문법: {target_grammar}
- 목표 어휘: {target_vocabulary}
- 연습문제 수: {exercise_count}
- 정답 포함: {include_answers}
- 해설 포함: {include_explanations}

난이도 지침:
- 초급: 짧은 문장, 일상 어휘, 현재/과거/미래 중심
- 중급: 연결어미와 이유/대조 표현 포함
- 고급: 추상적 주제, 의견 제시, 담화 구조 포함

출력 형식:
# 제목

## 학습 목표
- 2~3개 작성

## 읽기 전 질문
- 배경지식 활성화를 위한 질문 3개

## 읽기 지문
- 수준에 맞는 길이로 작성
- 목표 문법과 목표 어휘를 자연스럽게 포함

## 핵심 어휘
| 어휘 | 뜻 | 예문 |
| --- | --- | --- |

## 내용 확인 문제
- 참/거짓, 객관식, 단답형을 섞어서 작성

## 생각해 보기
- 말하기 또는 쓰기로 확장할 수 있는 질문 2개

## 정답 및 해설
- 각 문제의 정답과 간단한 근거
```

## 5. 문법 설명 자료 생성 프롬프트

### 목적

교사가 특정 문법 항목을 설명할 때 사용할 수 있는 설명, 예문, 오류 교정 자료를 생성한다.

### 사용자 프롬프트 템플릿

```text
다음 조건에 맞는 한국어 문법 수업자료를 만들어 주세요.

조건:
- 학습자 수준: {level}
- 목표 문법: {target_grammar}
- 주제 또는 상황: {topic}
- 목표 어휘: {target_vocabulary}
- 연습문제 수: {exercise_count}

출력 형식:
# 문법 항목: {target_grammar}

## 학습 목표
- 이 문법을 배운 후 학습자가 할 수 있는 일을 2개 작성

## 의미 설명
- 학습자 수준에 맞게 쉽게 설명

## 형태 정보
- 동사, 형용사, 명사와 결합하는 방법 설명
- 불규칙이나 주의점이 있으면 포함

## 사용 상황
- 이 문법을 주로 쓰는 상황 설명

## 예문
- 쉬운 예문 5개
- 목표 어휘를 자연스럽게 포함

## 비슷한 표현과 차이
- 학습자가 헷갈릴 수 있는 표현이 있으면 간단히 비교

## 자주 하는 오류
| 잘못된 문장 | 수정한 문장 | 설명 |
| --- | --- | --- |

## 연습문제
- 형태 바꾸기, 문장 완성, 상황에 맞게 말하기 문제 포함

## 정답 및 해설
- 각 문제의 정답과 간단한 설명

## 교사용 팁
- 설명할 때 주의할 점과 추가 활동 아이디어
```

## 6. 어휘 활동 생성 프롬프트

### 목적

특정 주제의 핵심 어휘를 학습하고 복습할 수 있는 활동지를 생성한다.

### 사용자 프롬프트 템플릿

```text
다음 조건에 맞는 한국어 어휘 활동지를 만들어 주세요.

조건:
- 학습자 수준: {level}
- 주제: {topic}
- 목표 어휘: {target_vocabulary}
- 연습문제 수: {exercise_count}
- 정답 포함: {include_answers}

출력 형식:
# 어휘 활동지: {topic}

## 학습 목표
- 2개 작성

## 핵심 어휘
| 어휘 | 품사 | 쉬운 뜻 | 예문 |
| --- | --- | --- | --- |

## 어휘 분류 활동
- 비슷한 의미, 장소, 행동, 물건 등으로 분류하는 활동

## 빈칸 채우기
- 문맥에 맞는 어휘를 넣는 문제

## 연결하기
- 어휘와 뜻 또는 어휘와 상황을 연결하는 문제

## 말하기 활동
- 목표 어휘를 사용해서 짝과 대화하는 활동

## 정답
- 모든 문제의 정답
```

## 7. TOPIK 스타일 문제 생성 프롬프트

### 목적

TOPIK 형식과 유사한 연습문제를 생성하되, 실제 기출문제를 복제하지 않고 새로운 문제를 만든다.

### 사용자 프롬프트 템플릿

```text
다음 조건에 맞는 TOPIK 스타일 한국어 연습문제를 새로 만들어 주세요.
실제 기출문제를 복사하거나 변형하지 말고, 완전히 새로운 문제를 작성해 주세요.

조건:
- 시험 수준: {level}
- 주제: {topic}
- 목표 문법: {target_grammar}
- 목표 어휘: {target_vocabulary}
- 문제 수: {exercise_count}
- 정답 포함: {include_answers}
- 해설 포함: {include_explanations}

출력 형식:
# TOPIK 스타일 연습문제

## 문제 세트 설명
- 문제의 목표와 난이도 설명

## 문제
- 각 문제는 번호를 붙여 작성
- 객관식은 보기 4개를 제공
- 정답이 명확하도록 작성

## 정답 및 해설
- 정답
- 정답인 이유
- 오답이 틀린 이유를 간단히 설명

## 교사용 주의사항
- 학습자가 헷갈릴 수 있는 지점 설명
```

## 8. 수업 활동지 변환 프롬프트

### 목적

이미 생성된 자료를 교실 활동 중심 활동지로 변환한다.

### 사용자 프롬프트 템플릿

```text
아래 한국어 수업자료를 교실 활동지 형식으로 변환해 주세요.

조건:
- 학습자 수준: {level}
- 수업 시간: {lesson_duration}
- 활동 방식: 짝 활동과 전체 활동을 모두 포함
- 교사가 바로 사용할 수 있게 단계별로 작성

원본 자료:
{source_content}

출력 형식:
# 활동지 제목

## 활동 목표
- 2~3개 작성

## 준비물
- 필요한 자료 또는 도구

## 활동 1: 도입
- 예상 시간
- 교사 발화 예시
- 학습자 반응 예시

## 활동 2: 연습
- 예상 시간
- 진행 방법
- 짝 활동 질문 또는 카드

## 활동 3: 확장
- 예상 시간
- 말하기 또는 쓰기 과제

## 마무리
- 정리 질문
- 숙제 아이디어

## 교사용 팁
- 진행 시 주의할 점
```

## 9. 난이도 낮추기 프롬프트

### 목적

생성된 자료가 어렵다고 느껴질 때 어휘와 문장 구조를 단순화한다.

### 사용자 프롬프트 템플릿

```text
아래 한국어 수업자료를 더 쉬운 수준으로 바꿔 주세요.

조건:
- 목표 수준: {level}
- 어려운 어휘는 쉬운 어휘로 바꾸기
- 긴 문장은 짧게 나누기
- 복잡한 문법은 줄이기
- 학습 목표와 핵심 내용은 유지하기
- 연습문제의 정답이 바뀌면 정답도 함께 수정하기

원본 자료:
{source_content}

출력 형식은 원본과 동일하게 유지해 주세요.
```

## 10. 난이도 높이기 프롬프트

### 목적

생성된 자료를 더 높은 수준의 학습자에게 맞게 확장한다.

### 사용자 프롬프트 템플릿

```text
아래 한국어 수업자료를 더 높은 수준으로 바꿔 주세요.

조건:
- 목표 수준: {level}
- 더 자연스럽고 다양한 표현을 추가하기
- 적절한 연결어미와 담화 표현을 추가하기
- 주제에 대한 의견 표현을 포함하기
- 연습문제는 단순 확인 문제보다 추론, 의견, 적용 문제가 포함되게 하기
- 정답과 해설도 수정된 자료에 맞게 업데이트하기

원본 자료:
{source_content}

출력 형식은 원본과 동일하게 유지해 주세요.
```

## 11. 정답 검수 프롬프트

### 목적

생성된 문제와 정답의 일관성을 점검한다.

### 사용자 프롬프트 템플릿

```text
아래 한국어 수업자료의 문제, 정답, 해설을 검수해 주세요.

검수 기준:
1. 각 문제의 정답이 명확한지 확인합니다.
2. 객관식 문제에 정답이 2개 이상 존재하지 않는지 확인합니다.
3. 정답과 해설이 서로 일치하는지 확인합니다.
4. 학습자 수준에 비해 지나치게 어려운 문제를 표시합니다.
5. 오류가 있으면 수정안을 제시합니다.

원본 자료:
{source_content}

출력 형식:
# 검수 결과

## 전체 평가
- 통과 / 수정 필요 중 하나 선택

## 발견된 문제
| 위치 | 문제점 | 수정 제안 |
| --- | --- | --- |

## 수정된 정답 및 해설
- 필요한 경우에만 작성
```

## 12. 비용 절감용 짧은 생성 프롬프트

### 목적

초기 초안만 빠르게 생성하고, 사용자가 원하는 경우 세부 자료를 추가 생성한다.

### 사용자 프롬프트 템플릿

```text
다음 조건에 맞는 한국어 수업자료의 짧은 초안만 만들어 주세요.
긴 설명은 쓰지 말고, 교사가 방향을 확인할 수 있을 정도로만 작성해 주세요.

조건:
- 학습자 수준: {level}
- 주제: {topic}
- 자료 유형: {content_type}
- 목표 문법: {target_grammar}

출력 형식:
# 제목

## 학습 목표
- 2개

## 핵심 내용 요약
- 5줄 이내

## 포함할 어휘
- 8개

## 포함할 활동
- 3개

## 다음 단계 제안
- 자세한 지문 생성
- 문제 생성
- 활동지 변환 중 선택할 수 있게 제안
```

## 13. 구현 메모

- 긴 자료를 한 번에 만들기보다 초안 생성, 지문 생성, 문제 생성, 정답 검수로 단계를 나누는 것이 비용 관리에 유리하다.
- 문제 생성 후에는 정답 검수 프롬프트를 별도로 실행할 수 있게 설계한다.
- 모든 프롬프트에는 학습자 수준과 목표 문법을 명시적으로 포함한다.
- 출력 형식은 Markdown을 기본으로 하되, 저장과 후처리를 위해 JSON 변환 가능성을 고려한다.
- 실제 서비스에서는 사용자 입력이 지나치게 길거나 저작권 있는 교재 전문을 포함하지 않도록 제한한다.
