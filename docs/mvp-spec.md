# 한국어 교사용 수업자료 제작 도구 MVP 기능 명세서

## 1. 제품 개요

### 1.1 제품 가설

한국어 교사는 수업 준비 과정에서 읽기 지문, 대화문, 어휘 활동, 문법 설명, 연습문제, 정답 및 해설을 반복적으로 제작한다. AI를 활용해 이 과정을 단축하되, 교사가 결과물을 검토하고 수정할 수 있는 구조를 제공하면 수업 준비 시간을 줄이고 자료의 다양성을 높일 수 있다.

### 1.2 MVP 목표

MVP의 목표는 완전 자동 교재 제작이 아니라, 교사가 바로 검토하고 편집할 수 있는 수업자료 초안을 빠르게 생성하는 것이다.

핵심 검증 질문은 다음과 같다.

1. FriendliAI의 EXAONE 계열 모델로 한국어 교육 자료를 안정적으로 생성할 수 있는가?
2. 교사가 결과물을 실제 수업 준비에 활용할 만큼 품질이 충분한가?
3. 생성 1회당 비용이 유료 서비스로 운영 가능한 수준인가?
4. 사용자가 프롬프트를 직접 작성하지 않아도 폼 입력만으로 원하는 자료를 얻을 수 있는가?

### 1.3 대상 사용자

1차 대상 사용자는 외국인 학습자를 가르치는 한국어 교사다.

세부 사용자 유형은 다음과 같다.

- 세종학당, 대학 부설 어학당, 사설 학원 한국어 교사
- 온라인 한국어 튜터
- 한국어 교육 콘텐츠 제작자
- 기업 또는 기관의 외국인 직원 대상 한국어 교육 담당자

### 1.4 MVP 포지셔닝

MVP는 다음 문장으로 설명할 수 있다.

> 주제와 학습자 수준만 입력하면, 한국어 교사가 수업에 활용할 수 있는 읽기 지문, 대화문, 어휘, 문법 설명, 연습문제, 정답 및 해설 초안을 생성하는 AI 수업자료 제작 도구.

## 2. 핵심 사용자 시나리오

### 2.1 기본 수업자료 생성

교사는 초급 학습자를 위한 “카페에서 주문하기” 수업을 준비한다. 앱에서 수준, 주제, 자료 유형, 목표 문법, 문제 수를 입력하고 생성 버튼을 누른다. 앱은 대화문, 핵심 어휘, 문법 설명, 연습문제, 정답을 포함한 자료를 생성한다.

### 2.2 기존 결과물 난이도 조정

교사는 생성된 지문이 초급 학습자에게 어렵다고 판단한다. “더 쉽게” 버튼을 누르면 앱은 문장 길이와 어휘 수준을 낮추고, 문법 구조를 단순화한 버전을 생성한다.

### 2.3 수업용 활동지로 변환

교사는 생성된 대화문을 수업 활동지로 바꾸고 싶다. “활동지로 변환” 버튼을 누르면 앱은 짝 활동, 빈칸 채우기, 역할극 과제, 교사용 진행 팁을 포함한 형태로 변환한다.

### 2.4 정답 및 해설 추가

교사는 문제만 생성한 후, 별도로 정답과 해설을 추가한다. 앱은 각 문제의 정답, 오답 해설, 교사용 주의사항을 생성한다.

## 6. AI 연동 요구사항

### 6.1 모델 연동 원칙

AI 연동은 특정 모델에 직접 종속되지 않도록 LLM Adapter를 통해 처리한다. 1차 모델은 FriendliAI의 EXAONE 계열 모델을 사용하되, 추후 다른 모델로 교체하거나 fallback을 둘 수 있어야 한다.

### 6.2 LLM Adapter 책임

- 모델 provider 선택
- API key 관리
- 프롬프트 템플릿 렌더링
- 요청 파라미터 설정
- 응답 파싱
- 실패 시 재시도
- 토큰 사용량 기록
- 예상 비용 계산
- 에러 코드 표준화

### 6.3 권장 요청 파라미터

| 목적 | temperature | max tokens | reasoning |
| --- | ---: | ---: | --- |
| 짧은 예문 생성 | 0.5 | 낮음 | off |
| 읽기 지문 생성 | 0.6 | 중간 | off |
| 문제 생성 | 0.4 | 중간 | off 또는 low |
| 정답 검수 | 0.2 | 낮음 | low |
| 전체 수업안 생성 | 0.5 | 높음 | low 또는 medium |

실제 파라미터 이름과 reasoning 설정 방식은 FriendliAI API 문서의 현재 스펙에 맞춰 구현한다.

## 8. 품질 검수 기준

### 8.1 자동 검수 항목

MVP에서는 최소한 다음 항목을 체크한다.

- 필수 섹션 누락 여부
- 문제 수가 요청과 일치하는지 여부
- 정답 필드 존재 여부

- 출력 길이가 과도하게 긴지 여부

### 8.2 교사용 품질 평가 항목

베타 테스트에서는 각 생성물에 대해 다음 평가를 받는다.

| 항목 | 척도 |
| --- | --- |
| 한국어 자연스러움 | 1~5 |
| 수준 적합성 | 1~5 |
| 수업 활용성 | 1~5 |
| 문제 정확성 | 1~5 |
| 수정 필요 정도 | 1~5 |

## 10. API 초안

### 10.1 콘텐츠 생성

```http
POST /api/contents/generate
```

요청 예시:

```json
{
  "projectId": "project_123",
  "level": "초급",
  "contentType": "dialogue_lesson",
  "topic": "카페에서 주문하기",
  "targetGrammar": "-고 싶어요",
  "targetVocabulary": ["메뉴", "주문하다", "계산하다"],
  "exerciseCount": 5,
  "includeAnswers": true,
  "includeTeacherTips": true
}
```

응답 예시:

```json
{
  "contentId": "content_123",
  "status": "success",
  "markdown": "# 카페에서 주문하기...",
  "usage": {
    "model": "exaone",
    "inputTokens": 1200,
    "outputTokens": 2800,
    "estimatedCostUsd": 0.0025
  }
}
```

### 10.2 콘텐츠 수정

```http
POST /api/contents/{contentId}/revise
```

요청 예시:

```json
{
  "revisionType": "make_easier",
  "instruction": "초급 1단계 학습자도 이해할 수 있게 바꿔 주세요."
}
```

### 10.3 사용량 조회

```http
GET /api/usage/me?period=month
```

## 11. MVP 성공 기준

MVP는 다음 조건을 만족하면 다음 단계로 진행한다.

- 베타 사용자 5명 이상이 실제 수업자료 제작에 사용한다.
- 생성 결과의 평균 품질 평가가 5점 만점 중 4점 이상이다.
- 생성 결과 중 정답 오류가 있는 비율이 5% 이하이다.
- 평균 생성 시간이 15초 이하이다.
- 생성 1회당 평균 추정 비용이 목표 범위 안에 있다.
- 사용자가 프롬프트를 직접 수정하지 않아도 원하는 자료를 생성할 수 있다.
